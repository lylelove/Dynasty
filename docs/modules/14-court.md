# 14 朝廷（mixins/court.py）

对应源码：`dynasty/mixins/court.py` · 模型：`models.Minister` · 状态：`state.py` 朝廷段

## 职责

明制内阁 + 六部的纯自动演化：组建、老病致仕、政争去职、卒于任、递补拜相、
权臣事件、朝臣谥号，以及对国运的「相辅」修正。无玩家任命/罢免交互；
可点击官员姓名查看详情。

## 官职（11 职）

| 职位 | 员额 | 备注 |
|------|------|------|
| 首辅 | 1 | 致仕概率减半、78 强制（恋栈） |
| 次辅 | 1 | 首辅出缺时第一顺位 |
| 群辅 | 3 | 内部不分先后，界面统称「群辅」（`post_display`） |
| 六部尚书 | 6 | 吏户礼兵刑工 |

## Minister 字段（models.py）

独立于宗室 `Person`：无父系、爵位、子嗣，不进 `self.people`，不参与 prune。
关键字段：`entry_year`（**初仕**之年，进士/释褐，早于尚书/入阁）、
`post_since_year`（现职就任）、`death_age`（50–84，招募时保证大于现年）、
`retired`、`shihao`（身后追谥）、`quanchen`（本任内已记权臣纪事）。

招募逻辑：尚书约 42–55 岁、阁臣约 48–62 岁；初仕约 22–30 岁，至尚书
至少 15 年仕途、直接入阁至少 20 年。开国班底现职另回溯已任 1–8 年。
能力门槛：首辅 ≥6、次辅/群辅 ≥5、尚书 ≥3（升迁时抬升，不降）。
UI「仕途年数」= 现年 − 初仕年；「本任年数」= 现年 − 现职就任年。
历任任期文案不用绝对年号（开局 year=0 回溯为负），见 `format_term_years`。

## 年度流程（`gamemin_court`，在 `gamemin_family_shihao_titles` 之后调用）

1. **新君察相**（`_court_new_emperor_check`）：检测 `emperor_id` 变化；新君
   以约 18% 概率处置前朝首辅（权臣 50%）。去向不限于「罢」，可含削职、
   外放、谪贬、病免、致仕，权臣更易赐死/伏诛。
2. **老与卒 / 高年去位 / 中途去职**（`_court_aging_and_exit`）：
   - `age >= death_age` → 卒于任（约 8% 暴卒）；寻常卒后追谥。
   - 65 起致仕概率 `0.10 + 0.05*(age-65)`，75 强制（首辅减半 / 78 强制）；
     高年去向在致仕 / 病免 / 终养间分配。
   - **中途去职**（任满 2 年后，年基率约 4–7%，材庸/权臣/主弱加权）：
     丁忧、病免、劾罢、外放、罢、削职、谪贬、终养、偶赐死/伏诛等，
     仿史实政争与人事代谢，避免历任表几乎全是「升迁/致仕」。
3. **林下终老**（`_court_offstage_deaths`）：去职仍在世者计龄至天年并追谥；
   曾任首辅者记「薨于里第」纪事。
4. **递补链**（`_court_fill_vacancies`）：次辅升首辅 → 群辅（能力最高）升次辅 →
   尚书（能力最高）入阁 → 招资深者补尚书（42–55，已有仕途）；循环级联至无缺。
   极端情形下阁职直接招资深者（48–60），保证 11 职常满。
5. **权臣察验**（`_court_quanchen_check`）：帝能力 ≤3、首辅能力 ≥8、在任 ≥5 年
   时每年 25% 触发一次「威权震主」纪事（每任至多一次），国运 -1.5（保底 1，
   生死只在 `gamemin_dynasty` 结算）。

## 谥号（`_grant_minister_shihao`）

**寻常卒后多得谥**（在职卒或林下终老）。**伏诛 / 赐死** 多不获谥
（`EXIT_NO_SHIHAO`）。材具分池：

| 能力 | 池 |
|------|-----|
| ≥8 且 40% | 极品（文正/文贞/文成/文忠） |
| ≥6 | 常谥（文端、文定……） |
| <6 | 次等常谥（文勤、文恪……） |

一朝尽量不重谥（`used_minister_shihao`）；池尽则回退允许重用非常/常谥。

## 国运影响（dynasty_logic.py）

`dynasty_hp += (_court_avg_ability() - 5.0) * 0.06`——在职 11 人均能力对 5 的
偏差，每年约 ±0.24，为主衰减项的 10–20%。空表/未组建返回 5.0（零影响）。

## 各职历任账（`post_history`）

`{post: [{mid, name, ability, start_year, end_year, exit}]}`。
`exit` 码见 `EXIT_LABELS`（`court.py`），常见包括：

| 码 | 展示 | 典型触发 |
|----|------|----------|
| 卒 / 暴卒 | 卒于任 / 暴卒于任 | 任上寿终 |
| 致仕 / 病免 / 终养 | 致仕 / 以疾致仕 / 乞终养 | 高年或急流勇退 |
| 丁忧 | 丁忧去位 | 中途亲丧 |
| 罢 / 劾罢 / 削职 | 罢黜 / 为言官所劾罢 / 削职闲住 | 新君或政争 |
| 外放 / 谪贬 | 外放边郡 / 谪贬 | 调离中枢 |
| 赐死 / 伏诛 | 赐死 / 伏诛 | 重罪（多无谥） |
| 升迁 | 升迁 | 递补链上迁 |
| `""` | 在任 | 尚未卸任 |

开闭：`_open_post_term` / `_close_post_term` / `_remove_from_post`，按 `mid` 匹配。
升职时先闭旧职（exit=升迁）再开新职；**同年级联升迁**产生的 0 年中间职
（start==本年 且 exit=升迁）直接删除，不入历任表。

`shoufu_history` 与 `post_history["首辅"]` **为同一列表引用**（兼容国史提示词与下表）。
`get_minister_career(mid)` 汇总某人各职履历，供详情弹窗。

流入：朝廷 Tab、官员详情、历任弹窗、国史提示词【五、宰辅】节。

## 纪事口径

只记首辅级 + 名臣入阁（ability ≥8），约每局 20–40 条。事件 dict 与
`event_happen()` 同构（含 `emperor_id`），自动流入纪事表、结束弹窗与国史提示词。

## 重置

三处必须同步：`state.init_game_state()`、`dynasty_logic.gamemin_dynasty_new()`
（重开）。字段：`ministers`、`next_minister_id`、`court_posts`、`post_history`、
`shoufu_history`（= `post_history["首辅"]`）、`used_minister_shihao`、
`court_last_emperor_id`。重开另清 `d_time/d_event/d_emperor` 快照，防新朝
开国纪事带前朝年号。
