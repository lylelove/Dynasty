dynasty——王朝

这是一个纯 Python/PySide6 搭建的桌面小游戏（原基于 Vue+element）。

## 运行游戏

请确保已安装 Python 环境，然后执行以下命令安装依赖并运行：

```bash
pip install -r requirements.txt
python main.py
```

**提示：** 关于皇室和国亲生育力的设定，可以在 `main.py` 的 `gamemin_family_marriage_birth` 方法中进行调整：
- 修改 `chance` 变量（控制是否有后代的整体概率，当前皇帝 0.5、宗亲 0.2）
- 修改 `random.random() < 0.2` 处的阈值（成婚概率）与 `random.random() < 0.3` 处的阈值（无子者额外生育机会）
从而控制宗亲子嗣的数量和繁衍速度。

本版本新增：
- **显眼世系树状图**：皇室宗亲（Tab 4）以醒目的树状图展示**完整父系世系**——以开国皇帝为根，按代数向下嵌套：如宋王之子受封彭城郡王，则彭城郡王一脉直接挂在宋王之下。每个节点显示姓名（在位者标★今上）、称号、国别（封国，已崩皇帝归“皇室”）、状态（存活/已故/绝嗣）、谥号/庙号/尊号与代数。表格视图仍按国别（封国）分组，便于分类查阅。
- **谥号修正**：开国皇帝（emperor_id==1）固定用褒谥，杜绝“炀”等恶谥；其余皇帝仅当功过明显败坏（评分 < -4）时才用恶谥，平庸守成之君多用中性谥，减少恶谥泛滥。
- **风味化内容**：大幅扩充封号（亲王/郡王/国公……封国与封号名库）、公主/郡主/县主/乡主封号、皇帝庙号（高祖至威宗等）、各类谥号（皇帝/后妃/太子/亲王/公主），并新增**皇帝尊号**（如“圣神文武皇帝”，登基时随机组合），皇帝信息页展示尊号，树状图与史书均体现庙号/谥号。
- **提高生育率**：成婚概率由 10% 提升至 20%，皇帝生育概率 30%→50%、宗亲 10%→20%，且无子嗣者每年额外有 30% 机会再育一子。
- **减少绝嗣**：宗亲无子而终时，自动尝试“过继”兄弟之子（侄）为嗣，继承封国、延续国祚，从而避免标记“绝嗣”。
